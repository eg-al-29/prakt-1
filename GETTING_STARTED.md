Структура:

```bash

pip install -r requirements.txt

# Запуск тестов
pytest tests/ -v

# Запуск конкретного теста
pytest tests/test_request_processor.py::TestRequestProcessorSuccess::test_successful_request_creation -v
```

## Основная идея

**Исходный код (original.py)** - демонстрирует проблемы:
- Смешение 8 ответственностей в одном методе
- Жёсткие зависимости (new DatabaseClient, new SmtpClient и т.д.)
- Невозможно тестировать без реальной БД/почты
- SQL-injection уязвимость
- Hardcoded конфиги

**Целевой дизайн (contracts.py + request_processor.py)** - решает проблемы:
- Разделение на слои (доменная логика + инфраструктура)
- Контракты вместо конкретных реализаций
- Dependency Injection через конструктор
- Легко подменять зависимости в тестах
- Расширяемо: новые каналы, логирование, БД - без изменения RequestProcessor
